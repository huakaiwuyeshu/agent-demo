# API 接入示例文档

## 1. 签名规则

所有参与签名的业务参数必须先按参数名的字母序升序排列，再用 `key=value` 形式拼接，参数之间使用 `&` 连接。

签名前字符串必须使用 UTF-8 编码。

签名前不要把参数值做 URL 编码。需要 URL 编码的字段，应在签名完成后再编码传输。

`timestamp` 表示请求时间戳。生产环境建议服务端允许 5 分钟以内的时间偏差。

`appsecret` 只允许在安全凭证系统中查看、重置或发放，不允许在群聊、普通工单、模型上下文或普通日志中出现。

## 2. 回调规则

回调地址必须是服务端可访问的 HTTPS 地址。

如果第三方反馈没有收到回调，应优先确认回调地址、订单号、触发动作、回调响应状态码和服务端日志。

## 3. 字段说明

`appid` 是第三方应用标识，用于识别接入方和接入环境。

`order_id` 是业务订单号，同一接入方下必须唯一。

`timestamp` 是请求发起时间戳，用于防止重放请求。

`sign` 是按签名规则生成的签名结果，不参与签名前字符串拼接。
