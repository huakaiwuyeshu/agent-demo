### 3. 工作流程详解卡片（新增）

**位置**：底部辅助区域之后，反馈栏之前

**内容**：8 步工作流程
```
┌────────────────────────────────────────────────────┐
│ Schema 校验 + Skill 路由工作流程                     │
├────────────────────────────────────────────────────┤
│ 1️⃣ 意图识别                                         │
│   关键词匹配 → task_type = "signature_debug"        │
│                                                    │
│ 2️⃣ 字段提取                                         │
│   正则提取 → fields = {appid: "xxx", ...}          │
│                                                    │
│ 3️⃣ Schema 校验                                      │
│   必填字段 vs 已提取字段 → status = "needs_..."     │
│                                                    │
│ 4️⃣ 精准追问                                         │
│   "还缺少：接口路径、签名前字符串、请求时间"          │
│                                                    │
│ 5️⃣ 用户补充                                         │
│   多轮对话 → 字段累积 → 记忆系统合并                 │
│                                                    │
│ 6️⃣ 校验通过                                         │
│   字段齐全 → status = "executable" → 进入路由        │
│                                                    │
│ 7️⃣ Skill 路由                                       │
│   skills["signature_debug"] → 加载 SKILL.md        │
│                                                    │
│ 8️⃣ 调用 LLM                                         │
│   System Prompt = Skill SOP + 字段 → 排查结论       │
└────────────────────────────────────────────────────┘

✓ 核心优势：一次性收集完整信息 → 避免多轮试错 
            → 确保工具可执行 → 标准化 SOP 输出
```

**视觉效果**：
- 渐变背景（蓝 → 绿）
- 8 格网格布局（响应式）
- 底部优势总结（绿色高亮）
