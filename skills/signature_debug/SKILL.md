# 签名失败排查 Skill

## 适用场景

第三方反馈接口返回 `SIGN_INVALID`、签名错误、验签失败、鉴权失败等问题。

## 必要信息

- 接口路径
- appid
- 签名前原始字符串
- 错误码
- 请求时间

不要要求用户提供 appsecret。

## 排查 SOP

1. 检查参与签名的参数是否按参数名的字母序升序排列。
2. 检查签名前字符串是否和 API 文档规则一致。
3. 检查签名前字符串是否使用 UTF-8 编码。
4. 检查是否在签名前提前做了 URL 编码。
5. 检查 timestamp 是否超过服务端允许的时间偏差。
6. 检查 appid 是否属于当前环境。

## 输出口径

先说明最可能原因，再给出可执行的下一步动作。若信息不足，只追问阻塞项。
